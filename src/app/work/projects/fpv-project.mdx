## Overview

A hands-on personal engineering project involving the design, construction, and configuration of multiple FPV freestyle drones from the ground up. Born out of a passion for flying and a tight budget, this project became a deep dive into electronics, firmware programming, and hardware debugging — skills that bridge the gap between software development and real-world embedded systems.

## Key Featuresnp

- **Custom Hardware Builds**: Assembled multiple FPV freestyle drones by combining components from broken and salvaged parts, keeping costs low while maximising performance.
- **Betaflight Firmware Configuration**: Programmed each drone using Betaflight, utilising its CLI (Command Line Interface) — a text-based command parser built on C — to fine-tune input/output parameters and flight characteristics.
- **Soldering & 3D Printing**: Hands-on fabrication including soldering electronic components and 3D printing replacement or custom parts to repair and iterate on builds.
- **Debug & Repair Workflow**: Developed a systematic approach to diagnosing faults after crashes — from identifying failed ESCs and motors to reconfiguring PID values and testing flight stability.

## Technologies Used

- **Betaflight**: Flight controller firmware used to configure and tune drone behaviour via CLI commands.
- **C (via Betaflight internals)**: Indirect exposure to C programming through Betaflight's underlying codebase and CLI tooling.
- **Soldering & Electronics**: Hands-on wiring of ESCs, flight controllers, VTX units, cameras, and motors.
- **3D Printing**: Custom bracket and part fabrication for repairs and modifications.
- **CLI Debugging**: Text-based diagnostics and parameter tuning through the Betaflight command line.

## Challenges and Learnings

The biggest challenge was learning to fly — crashes were frequent in the early stages, which meant constantly repairing and rebuilding. Rather than being a setback, this became the core learning opportunity. Each repair required diagnosing whether the fault was hardware (broken solder joint, burnt ESC, bent motor shaft) or software (misconfigured PIDs, incorrect motor mapping). Working with a limited budget forced creative problem-solving: salvaging usable components from broken frames and repurposing them into functional builds. This developed a strong practical understanding of how hardware and firmware must be configured to work in unison.

## Outcome

Successfully built and flew several FPV freestyle drones, progressing from a complete beginner to confidently diagnosing, repairing, and tuning drone builds. The project solidified core skills in embedded electronics, firmware configuration, and hardware debugging — directly transferable to firmware development and prototype testing roles in engineering teams.
